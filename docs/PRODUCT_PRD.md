# FreeRAG PRD v2

更新时间：2026-05-23

## 一句话定位

FreeRAG 是一个 Mac 本地原材料收集器。它以极低操作成本收集屏幕、剪贴板、语音等零散信息，并把它们整理成 LLM 友好的本地语料；MyRAG skill 负责后续检索、多视角深挖、证据校准和处理结果沉淀。

FreeRAG 不做完整知识库、不做笔记软件、不做剪贴板历史工具，也不在采集端自动调用网络模型。它是“原材料入口 + LLM 处理层”的组合产品。

## 产品目标

用户需求：

> 在以 Mac 为核心的日常工作和生活中，低成本、顺滑地收集零散信息，让后续 LLM 可以可靠地找回、深挖和复用。

目标体验：

- 用户不需要每次想“我要整理资料了”才开始操作。
- App 常驻、轻、安静，不打断当前工作。
- 原材料落盘结构稳定，LLM 能直接读。
- MyRAG 不只检索关键词，而是把有限 RAG 横过来竖过去读，提取更高密度信息。
- 深挖结束后收束到明确出口：存工作日志、结合已有项目、或进入外部调研。

## 产品架构

```text
FreeRAG.app
  -> 本地采集
  -> ~/Documents/Corpus/
       _index.json
       _library.json
       README_FOR_LLM.md
       screen/
       clipboard/
       voice/
       processed/

MyRAG skill
  -> 检索 _library.json / _index.json
  -> 阅读 raw entry + processed entry
  -> 多视角深挖
  -> 写回 processed/<entry_id>/
  -> 必要时生成工作日志 / 结合项目 / 外部调研
```

## 范围边界

### FreeRAG App 负责

- 常驻菜单栏和浮窗。
- 屏幕连续采样。
- 单次截图。
- 剪贴板文字和图片自动收集。
- 本地语音 WAV 录制。
- 权限准备。
- 登录时自动启动。
- 简单原材料库浏览。
- 写入 LLM 友好的目录、索引和上下文文件。

### MyRAG Skill 负责

- 检索本地语料。
- 读取原材料和已有处理结果。
- 多视角深挖。
- 证据校准：事实、材料主张、意图/愿望、推断。
- 反方攻击：识别最脆弱证据、假设和缺口。
- 写回 `processed/<entry_id>/`。
- 在深挖后收束到三个出口。

### 明确不做

- FreeRAG 不自动 OCR、自动转写或自动联网。
- FreeRAG 不做复杂知识库管理。
- FreeRAG 不做大而全的 clipboard manager。
- MyRAG 不无限分析，不自作主张变成项目管理系统。
- MyRAG 不把外部调研事实和本地 RAG 结论混在一起。

## 采集能力

### 剪贴板收集

目标：用户复制文字或普通图片时，后台自动保存有价值碎片。

当前策略：

- 文字 <= 1 MB 自动保存。
- 图片 <= 32 MP 且源数据 <= 96 MB 自动保存。
- 重复内容通过 hash 抑制。
- 文件列表、超大 payload、富格式私有数据暂不收集。

输出：

```text
clipboard/<entry_id>/
  content.md
  image.png         # 图片条目才有
  _meta.json
  llm_context.md
```

### 屏幕连续采样

目标：保存用户正在看的窗口/屏幕变化，作为后续复盘和 LLM 阅读证据。

当前策略：

- 启动后采样当前前台窗口，无法识别窗口区域时退回屏幕区域。
- 使用视觉签名过滤近似重复画面。
- 停止后生成 `captures.json` 和 `stitched/storyboard_XX.png`。
- 这不是视频录制；它是屏幕证据采样。

输出：

```text
screen/<entry_id>/
  001.png
  002.png
  captures.json
  stitched/storyboard_01.png
  _meta.json
  llm_context.md
```

### 单次截图

目标：一键保存当前上下文。

当前策略：

- 使用与连续采样相同的目录协议。
- 不弹预览、不要求确认。
- 结果进入全局索引。

### 语音笔记

目标：快速保存口头上下文，后续由 LLM/语音工具转写或整理。

当前策略：

- 16 kHz mono WAV。
- 支持暂停、继续、停止。
- FreeRAG 只保存录音和占位转写，不自动转写。

输出：

```text
voice/<entry_id>/
  recording.wav
  transcript.md
  _meta.json
  llm_context.md
```

## MyRAG 深挖协议

MyRAG 默认中文输出。它面对的不是“知识库成品”，而是用户本地收集的 raw 原材料。

默认流程：

1. 明确问题。
2. 判断场景。
3. 检索候选语料。
4. 建立材料清单。
5. 对散乱语料做主题/项目归拢建议。
6. 选择 4-8 个阅读视角。
7. 分视角逐轮深读。
8. 证据校准。
9. 反方攻击。
10. 交叉合并。
11. 写回处理层。
12. 回答用户。
13. 收束出口。

主题/项目归拢：

- 当用户要整理一批乱语料时，MyRAG 先把条目语义归成 3-8 个候选事项。
- 这些候选事项可以是正式项目，也可以是产品事项、生活事项、研究主题、偏好线索、问题池或待办池。
- 每个候选事项需要给出名称、一句话说明、代表条目、归类证据、置信度和建议出口。
- 用户确认前，不把候选事项写入长期项目。
- 归拢只过滤 FreeRAG 的协议/运营层生成物，不过滤真实语料里的 app 名、项目名、公司名或技术名。

通用视角：

- L01 原始事实镜头
- L02 时间与情境镜头
- L03 人事物关系镜头
- L04 意图与需求镜头
- L05 结构与机制镜头
- L06 细节与数值镜头
- L07 情绪与偏好镜头
- L08 矛盾与缺口镜头
- L09 模式与反例镜头
- L10 决策与取舍镜头
- L11 行动与机会镜头
- L12 沉淀与复用镜头
- L13 反方攻击镜头

收束出口：

1. **存工作日志**：生成 Markdown，默认进入 `processed/_worklogs/`。
2. **结合已有项目**：先问项目和路径，再合并项目资料。
3. **外部调研**：说明本地 RAG 缺口，再进入 websearch，并分开标注本地结论和外部事实。

## UI 原则

### 浮窗

浮窗只负责即时采集。

Idle 主动作：

- 连续收集
- 单次截图
- 录音

Idle 二级动作：

- 打开语料库
- 隐藏浮窗

这两个动作必须视觉弱化，不能和主采集动作争抢注意力。保留它们是因为用户明确需要快速打开和快速隐藏。

Active 状态：

- 连续收集：显示有效帧数、暂停、完成。
- 语音：显示时间、暂停、完成。
- 暂停：显示计数/时间、继续、完成。

视觉规则：

- SF Symbols。
- 同一字重和视觉尺寸。
- 按下态必须在鼠标释放前保持。
- 不使用 emoji。
- 不使用虚假的“玻璃高光”；要么原生材料，要么干净扁平。
- 每个可见元素必须有功能价值。

### 菜单栏

菜单栏负责 app 管理和备用入口：

- 显示浮窗。
- 连续收集 / 单次截图 / 录音。
- 暂停 / 继续 / 完成当前收集。
- 语料库。
- 在 Finder 中打开语料库。
- 设置。
- 退出。

### 设置

设置负责：

- 权限准备。
- 权限状态。
- 登录时自动启动。
- 授权后重启提示。

不要把“登录时启动”放成菜单栏顶层按钮。

## 权限体验

FreeRAG 需要：

- 屏幕录制：截图和连续屏幕证据。
- 辅助功能：读取前台窗口标题和区域。
- 麦克风：录音。

原则：

- 首次启动集中引导权限。
- 尽可能提前触发系统授权，避免用户真正录音/截图时才被弹窗打断。
- macOS 需要手动设置或重启时，明确提示。
- 如果权限缺失，相关动作打开设置，不静默失败。

## 数据协议

根目录：

```text
~/Documents/Corpus/
```

关键文件：

- `_index.json`：旧版兼容全局索引。
- `_library.json`：当前 raw 原材料索引。
- `README_FOR_LLM.md`：给 LLM 的目录说明。
- `processed/`：LLM 处理层。

处理层推荐结构：

```text
processed/<entry_id>/
  brief.md
  deep_read.md
  facts.json
  citations.md
  open_questions.md
```

## 当前符合度

已符合：

- Native Swift/AppKit app。
- 菜单栏 + 浮窗。
- 浮窗跨 Spaces / full-screen auxiliary。
- 图标化 HUD，主动作清晰。
- 剪贴板文字/图片自动保存。
- 屏幕采样和单次截图落盘。
- 语音 WAV 落盘。
- `_index.json`、`_library.json`、`llm_context.md`、`README_FOR_LLM.md`。
- 原材料库轻量浏览。
- MyRAG 中文优先、多视角深挖、证据校准、三出口收束。
- MyRAG 支持 `--suggest-projects` 辅助读取最近语料，供 LLM 做主题/项目归拢。

部分符合 / 需要继续打磨：

- 屏幕采样当前是固定间隔 + 去重，尚未实现真正的稳定退避采样。
- PRD 早期写过“拼接重叠长图”，当前实现实际是 storyboard，不是无缝长图；当前产品定义应以 storyboard 为准。
- 浮窗捕获前没有显式 hide/show 避免自捕获；当前依赖窗口区域和系统截图行为，需要继续实测。
- 登录时启动当前使用 LaunchAgent plist；更原生方向可以后续迁移到 `SMAppService`。
- Library UI 是轻量 raw browser，够用但还不够精致。
- MyRAG 写工作日志的实际执行流程已经定义，但工具脚本还没有专门的 `--write-worklog` 命令。

## 下一步建议

优先级 P0：

- 实测截图、连续采样、录音、剪贴板图片在真实使用中的权限和落盘表现。
- 修正任何会导致“授权了但仍显示未生效”的状态问题。
- 用真实语料跑一次 MyRAG 深挖，验证 processed 文件是否足够 LLM-friendly。
- 用最近 40 条真实语料跑一次 MyRAG 归拢，验证候选事项是否符合用户直觉。

优先级 P1：

- 为 MyRAG 增加 `--write-worklog` 辅助命令。
- Library UI 继续精修：空状态、筛选、详情排版、打开 processed。
- 屏幕采样加入稳定退避。
- 权限状态增加更明确的“已授权但需重启”状态。

优先级 P2：

- 后续 DMG 包装。
- 公证/签名流程。
- 更原生的登录项实现。
