FreeRAG Mac 本地语料入口 + MyRAG skill

把 Mac 上的零散信息，顺手变成 LLM 可深挖的本地语料。

FreeRAG 负责低成本收集屏幕、剪贴板和语音原材料；MyRAG 负责检索、多视角深读、证据校准和处理结果沉淀。它不是笔记软件，而是面向 AI 的原材料工坊。

产品边界

FreeRAG 的价值不在于把采集端做成复杂知识库，而在于把原材料收得足够顺，再让 LLM 在需要时处理得足够深。

FreeRAG App

安静常驻，负责即时采集和本地落盘。它尽量不解释、不打扰、不联网。

语料目录

稳定目录协议，保留原始文件、索引、上下文和 processed 处理层。

MyRAG Skill

中文优先，检索本地 RAG，按多个镜头深读，再把结果写回处理层。

采集能力

四个入口覆盖多数日常零散信息：正在看的屏幕、瞬间截图、复制内容、口头上下文。

连续收集

对当前窗口/屏幕做证据采样，过滤近似重复画面，停止后生成 storyboard。

单次截图

一键保存当前上下文，使用同一语料协议，不弹预览。

剪贴板

后台保存合理大小的文字和图片，抑制重复，忽略超大 payload。

语音

本地保存 16 kHz mono WAV 和占位 transcript，后续再转写。

原材料库

轻量浏览 raw entries，快速打开条目或 Finder 目录。

权限准备

集中处理屏幕录制、辅助功能、麦克风和登录时自动启动。

语料协议

目录结构对人类保持可读，对 LLM 保持稳定。FreeRAG 只写 raw，MyRAG 写 processed。

MyRAG 深挖方式

MyRAG 不停留在关键词召回。面对散乱语料时，它先归拢出候选事项，让用户确认；确认后再选择少数相关镜头反复读材料，最后做证据校准和收束。

通用镜头

  • 事实、时间、关系、意图
  • 结构、细节、偏好、矛盾
  • 模式、取舍、行动、复用
  • 反方攻击：最脆弱证据和假设

证据校准

  • 可验证事实
  • 材料主张
  • 意图 / 愿望
  • 我的推断
主题归拢 找事实 复盘 做决策 提取偏好 排错 生活碎片

深挖后的三个出口

MyRAG 不无限扩展。它把本地语料挖透后，只做明确收束。

存工作日志

当这次思考本身有价值，生成 Markdown，保存到 processed/_worklogs。

结合已有项目

需要挂到项目时，先问项目和路径，再读取项目资料做合并。

外部调研

本地 RAG 缺最新事实或第三方证据时，再进入 websearch，并分开标注来源。
