# FreeRAG 设计规范

FreeRAG 是一个小工具，但不能像临时脚本。它应该像一个安静、精确、随手可用的 Mac 原生工具。

## 产品表面

FreeRAG 只有三个主要表面：

1. **浮窗**：即时采集。
2. **菜单栏**：app 管理和备用命令。
3. **设置 / 原材料库**：权限、启动项和轻量浏览。

## 浮窗规则

Idle 主动作：

- 连续收集。
- 单次截图。
- 录音。

Idle 二级动作：

- 打开语料库。
- 隐藏浮窗。

二级动作必须弱化，并通过分隔线和主采集动作分组。保留它们是为了满足快速打开和快速隐藏，不是为了把浮窗变成工具栏。

Active 状态：

- 连续收集：有效帧数、暂停、完成。
- 语音：时间、暂停、完成。
- 暂停：当前计数/时间、继续、完成。

视觉规则：

- 使用 SF Symbols。
- 图标保持统一尺寸、字重和光学比例。
- 不使用 emoji。
- 不使用生成位图做界面按钮。
- 不做假的玻璃高光；使用原生 material 或干净扁平面板。
- 按钮按下态要持续到鼠标释放。
- 没有装饰性状态点；每个可见元素都必须回答用户问题或触发动作。

## 菜单栏规则

菜单栏包含：

- 显示浮窗。
- 连续收集、单次截图、录音。
- 暂停、继续、完成当前收集。
- 语料库。
- 在 Finder 中打开语料库。
- 设置。
- 退出。

不要把“登录时自动启动”放在菜单栏顶层；它属于设置。

## 设置规则

设置页包含：

- 屏幕录制权限。
- 辅助功能权限。
- 麦克风权限。
- 全部准备。
- 刷新。
- 登录时自动启动。
- 重启提示。

设置页要像产品面板，不像调试窗口。所有文案要短、明确、对齐。

## 原材料库规则

原材料库不是完整知识管理系统。它只需要帮助用户偶尔查看和打开 raw 条目：

- 类型筛选。
- 搜索标题、摘要、上下文。
- 打开条目。
- 在 Finder 中显示。
- 查看 `llm_context.md`。

不要把 processed 管理做复杂。主读者是 LLM，不是人类。

## App Icon

App icon 规则：

- 简洁抽象。
- 无文字。
- 小尺寸可读。
- 视觉语言和 app 内部一致。
- 表达“信息碎片进入本地容器”的隐喻。
